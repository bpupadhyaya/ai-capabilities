On-Device Intelligence

Mobile & On-Device Models

LLMs that run on smartphones, tablets, and edge devices — entirely without internet. Private by design. Functional in the most remote and restricted environments.

AGI Roadmap WebLLM Demo
Mobile inference rules of thumb: Minimum viable LLM on mobile requires approximately 1.5 GB RAM (1B-2B params at INT4). For a useful chat model, target 3–4 GB RAM (3B–4B params). Most modern smartphones (2022+) have 6–12 GB RAM. Storage for a Q4 quantized 1B model is roughly 0.7 GB; a 4B model is roughly 2.5 GB. Neural processing units (NPUs) on Apple A-series and Qualcomm Snapdragon chips can run INT4/INT8 inference at 2–5x lower power than the CPU alone.

Device Compatibility

🤖
Android
Pixel 6+ (Tensor G1+), Samsung Galaxy S21+, any device with 6 GB+ RAM. Use MLC LLM, MediaPipe, or llama.cpp via Termux. Pixel 9 Pro has Google AI on-device APIs.
🍎
iOS / iPadOS
iPhone 12+, iPad with A14+. Use MLC LLM (Metal), Core ML (via Apple's Foundation Models API), or llama.cpp iOS builds. Apple Intelligence uses on-device models on A17 Pro / M-series.
🍓
Raspberry Pi & SBCs
Raspberry Pi 5 (8 GB RAM) runs Llama 3.2 3B at ~2–3 tok/s via llama.cpp. Pi 4 (4 GB) handles 1B models comfortably. Orange Pi, Rock 5B, and Radxa boards are also viable.
⚙
Embedded / Jetson
NVIDIA Jetson Orin NX (16 GB) runs 7B models at real-time speeds. Jetson Nano handles 1B–3B. For MCUs, look at TensorFlow Lite Micro and ONNX Runtime Mobile for encoder models.

Mobile Inference Runtimes

🦙
llama.cpp
The universal runtime. Runs on Android (via Termux or NDK), iOS (via Swift bindings), Raspberry Pi, and desktop. Supports GGUF quantization. Widest model compatibility.
🚀
MLC LLM
Machine Learning Compilation from CMU. Pre-compiled models for iOS (Metal), Android (Vulkan/OpenCL), and WebGPU. Fastest throughput on mobile GPUs. Official iOS/Android apps available.
🔦
ExecuTorch
Meta's on-device inference library. Integrates with PyTorch. Supports Llama 3.2 with XNNPACK (CPU), Metal (iOS), and Vulkan (Android) backends. Available as Swift/Java SDK.
🍏
Core ML
Apple's first-party inference framework. Uses ANE (Neural Engine) for maximum efficiency on Apple Silicon. Best option for iOS production apps. Gemma and Llama have Core ML conversion tools.
🎯
TensorFlow Lite
Google's mobile runtime. MediaPipe uses TFLite under the hood for Gemma 3n. Wide Android ecosystem support. GPU delegate accelerates transformer models.
🔷
ONNX Runtime Mobile
Cross-platform. Encoder models (Whisper, embedding models) run efficiently. Full generative model support via ONNX Runtime GenAI. Works on iOS and Android.
